# 学籍管理数据分析工具集

> 高校管理者自己动手搭建的数据分析系统。六个Python脚本，零外部依赖。

## 这是什么

把散落在Excel、Word、教务系统里的学生数据，用SQLite串成一条完整的生命周期线：
录取 → 注册 → 在校 → 异动（退学/休学/复学/转专业）

## 能干什么

- **录取漏斗**：4年招生数据，专业热度变化趋势一目了然
- **留存率排名**：各学院报到率、留存率横向对比
- **异动追踪**：谁走了、为什么走、从哪个学院走的
- **报表生成**：填表不用翻Excel，一条查询出结果

## 文件说明

| 文件 | 作用 |
|------|------|
| `rebuild_roster.py` | 从在校名单重建数据库 |
| `import_2025_xueji.py` | 导入2025年学籍异动 |
| `import_2026_xueji.py` | 导入2026年学籍异动 |
| `data_tools.py` | 通用查询/统计工具 |
| `init_db.py` | 建表初始化 |
| `数据字典与业务规则.md` | 字段说明、学院映射、统计规则 |
| `2025级新生留存率分析.md` | 留存率排名分析报告 |

## 技术栈

- Python 3 + SQLite（无需安装服务器）
- 跨表关联：学号/考生号作为统一钥匙
- 跨库查询：`ATTACH DATABASE` 合并多个 .db 文件

## 数据安全

- ❌ 本仓库**不含任何学生数据**（.db文件已排除）
-  所有分析数据导入前已脱敏（删除身份证号、手机号等）
-  学校名称和个人路径已替换为占位符

## 方法论

核心原则：**数据架构不是设计出来的，是问题逼出来的。** 先建最小可用表，等真实问题来找你——每个新字段背后都有一个你必须回答的问题。
